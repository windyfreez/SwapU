
# SwapU 云市集

**SwapU 云市集**是一款专为校园场景设计的二手交易平台。项目旨在解决传统校园二手交易中存在的**信息碎片化、流程不透明、交易不便捷**等核心痛点，打造一个规范、高效、诚信的校园闲置生态圈。

## 一、 技术栈与架构体系

* **后端框架**：Spring Boot
* **持久层框架**：MyBatis
* **数据库**：MySQL (业务数据存储) + Redis (高性能缓存/数据削峰)
* **前端框架**：Vue.js (响应式交互)
* **即时通讯**：WebSocket (全双工实时通信)

---

## 二、 核心业务架构模块

为了系统化地解决校园二手交易难题，我们将业务逻辑拆解为以下五个协作模块：

### 1. 用户身份与资产管理

打造基于校园身份的信任基石。

* **账户体系**：支持用户注册、登录，并提供密码修改与基础信息更新接口，保障账户安全。
* **个人画像与查询**：支持获取当前用户详细信息，方便在交易过程中进行实名认证与信用展示。
* **收藏夹服务**：为用户提供“心愿清单”功能，支持快捷添加或移除心仪闲置。

### 2. 商品全生命周期管理

解决交易信息不集中、发布不规范的问题。

* **精准分类与浏览**：通过分类列表进行筛选，并提供热门商品推荐及多维度分页查询。
* **商品标准化维护**：支持商品发布、详情编辑、下架及删除操作，规范二手商品展示信息。
* **透明检索**：用户可轻松查看个人发布历史或按 ID 定位特定闲置，大幅提升寻找商品的效率。

### 3. 订单交易闭环

将非正式的“私下交易”转化为标准化的“平台订单”。

* **交易流程**：通过线上创建订单、支付及取消订单，确保交易意向明确。
* **闭环物流与执行**：提供商家确认接单、订单发货，以及买家确认收货的闭环链路，保障货款安全。
* **统计分析**：提供订单详情查询及系统层面的订单统计信息，为运营提供数据支持。

### 4. 实时通讯 (IM) 模块

解决买卖双方沟通不及时、社交隐私泄露问题。

* **会话与历史管理**：支持获取聊天会话列表及历史消息记录，随时查阅沟通内容。
* **实时交互**：基于 WebSocket 实现即时消息推送，并具备“消息已读”状态同步功能，保障沟通实时性。

### 5. 辅助服务

* **统一资源服务**：提供标准化的文件上传接口（图片、附件），确保二手商品展示图的标准化存储与加载。

## 三、 系统性能优化与架构设计

为应对校园高峰时段的访问压力，SwapU 采用了深度优化方案，确保系统在海量闲置浏览请求下依然流畅。

### 1. 热门商品高并发缓存策略

针对闲置商品浏览量高频写入与热点排行查询的性能瓶颈：

* **写入削峰**：利用 Redis 缓冲浏览量增量，定时批量回写 MySQL，避免数据库写锁竞争。
* **热点预热**：定时计算 Top 20 热门闲置并预置入 Redis，将接口响应速度从毫秒级提升至 5ms 以内。

### 2. 首页智能推荐的“动态多样性”算法

为破解传统二手平台“旧货长期霸屏，新品无人问津”的痛点：

* **分区随机重排**：将商品池拆分为“热门区”与“常规区”，并在分区内执行随机 Shuffle。
* **业务效果**：确保高热度闲置获得曝光的同时，给新发布的闲置商品公平的展示权重，显著提升首页的探索新鲜感。

## 四、 开发与协作指南

> * **安全校验**：所有涉及 User 和 Order 的核心接口，务必在 Header 中进行身份 Token 校验，确保交易真实可控。
> * **缓存优先原则**：热门商品查询严禁直接对数据库执行全表排序，务必优先从 Redis 获取。
> * **实时通信说明**：建议前端接入 WebSocket 协议，实现“即时聊天”的顺滑体验。
